# Plans

Use lightweight inline plans for small changes. For larger work, add an execution plan under `docs/plans/active/` and move it to `docs/plans/completed/` when done.

Each checked-in plan should include:

- Goal and non-goals
- Affected files or modules
- Validation commands
- Progress log
- Decisions that future agents should preserve

Do not add large speculative plans. Prefer short, current plans that help the next agent continue from repository-local context.
